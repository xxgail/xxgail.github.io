php笔记

PHP入门笔记，课程来源慕课网

入门篇:

  1. 输出echo与字符串之间至少要有一个空格；
    echo在输出布尔值时，TRUE输出1，FALSE什么也不输出

  2. 点(.)表连接

  3. 变量名必须通过$符号开头，使用变量需要加该符号；
    var_dump($变量名)”函数可以将变量的数据类型显示出来

  4. 变量名以字母或下划线开头，可以包含汉字；注意区分大小写

  5. “= =”的优先级高于“=”
  6. 单双引号的区别:
    当双引号中包含变量时，变量会与双引号中的内容连接在一起；
    当单引号中包含变量时，变量名会被当做字符串输出

  7. Heredoc结构形式用来输出很长的字符串

  8. 读取文件

  9. 常量一般用大写字母字符串表示 。可以不加$。定义常量define(“常量名“，值)。

  10. 常见的系统常量有：
    （1）FILE :php程序文件名。它可以帮助我们获取当前文件在服务器的物理位置。
    （2）LINE :PHP程序文件行数。它可以告诉我们，当前代码在第几行。
    （3）PHP_VERSION:当前解析器的版本号。它可以告诉我们当前PHP解析器的版本号，我们可以提前知道我们的PHP代码是否可被该PHP解析器解析。
    （4）PHP_OS：执行当前PHP版本的操作系统名称。它可以告诉我们服务器所用的操作系统名称，我们可以根据该操作系统优化我们的代码。

  11. 获取常量的另一种方法:constant(常量名称)

  12. 判断常量是否存在:define(常量名称):存在返回TRUE，不存在返回FALSE。
  13. “&”：引用赋值，意味着两个变量都指向同一个数据。原变量改变，该变量也会变。
  14. PHP中的比较运算符多了这两个:
    全等于 : $a === $b，即$a 等于 $b,且它们的类型相等，返回TRUE；
    非全等 : $a !== $b，即 $a 不等于 $b或者它们的类型不同，返回TRUE。
  15. xor逻辑异或:有且仅有一个为TRUE，则返回TRUE。

  16. （1）连接运算符(“.”)：它返回将右参数附加到左参数后面所得的字符串。
    （2）连接赋值运算符(“.=”)：它将右边参数附加到左边的参数后。

  17. @字符可以省略错误信息。但是不能把它放在函数或类的定义之前，也不能用于条件结构例如if和foreach等。

  18. 在switch语句中加break;的作用是组织代码进入下一个case中执行。

  19. foreach循环语句:
    （1）只取值，不取下标

    （2）同时取下标和值

  20. 关于zend studio快捷键的用法:
    Ctrl+Alt+↓ 复制
    Ctrl+/ 行注释
    Ctrl+shift+/ 块注释
    Alt+/ 提示助手
    Ctrl+f 调出查找面板
    Ctrl+h 调出全文检索面板
    Ctrl+w 关闭当前文件
    Ctrl+shift+l 查看快捷键列表

  1. PHP中函数的定义方式:
    3.1.使用关键字“function”开始
3.2.函数名可以是字母或下划线开头：`function name()`
3.3.在大括号中编写函数体：

  2. 返回语句return;会立即中止函数的运行，并且将控制权交回调用该函数的代码行。

  3. 可以通过改变一个变量的值来实现调用不同的函数。

    可变函数也可以用在对象的方法调用上。

  4. 构造函数: __construct() ，具有构造函数的类，会在每次对象创建的时候调用该函数，因此常用来在对象创建的时候进行一些初始化工作。
    在子类中如果定义了__construct 则不会调用父类的 __construct，如果需要同时调用父类的构造函数，需要使用parent::__construct()显式的调用。

  5. 析构函数: 使用__destruct()进行定义，析构函数指的是当某个对象的所有引用被删除，或者对象被显式的销毁时会执行的函数。

  6. PHP中的重载指的是动态的创建属性与方法，是通过魔术方法来实现的。属性的重载通过set，get，isset，unset来分别实现对不存在属性的赋值、读取、判断属性是否设置、销毁属性。

    方法的重载通过__call来实现，当调用不存在的方法的时候，将会转为参数调用__call方法，当调用不存在的静态方法时会使用__callStatic重载。

  7. (1)正则表达式中常用的元字符:

    • \ 一般用于转义字符
    • ^ 断言目标的开始位置(或在多行模式下是行首)
    • $ 断言目标的结束位置(或在多行模式下是行尾)
    • . 匹配除换行符外的任何字符(默认)
    • [ 开始字符类定义
    • ] 结束字符类定义
    • | 开始一个可选分支
    • ( 子组的开始标记
    • ) 子组的结束标记
    • ? 作为量词，表示 0 次或 1 次匹配。位于量词后面用于改变量词的贪婪特性。 (查阅量词)
    • *量词，0 次或多次匹配
    • +量词，1 次或多次匹配
    • { 自定义量词开始标记
    • } 自定义量词结束标记
      (2)只能在方括号内使用的有：
    • \ 转义字符
    • ^ 仅在作为第一个字符(方括号内)时，表明字符类取反。其中^在方括号外面，表示断言目标的开始位置，但在方括号内部则代表字符类取反，方括号内的减号-可以标记字符范围。
    • -标记字符范围

  8. 如果提供了参数 matches ，它将被填充为搜索结果。$matches[0]将包含完整模式匹配到的文本。$matches[1] 将包含第一个捕获子组匹配到的文本，以此类推。

  9. 贪婪模式(+):在可匹配与可不匹配的时候，优先匹配。
    懒惰模式(？):在可匹配与可不匹配的时候，优先不匹配。
    可指定长度:{长度}。

  10. preg_match只能匹配一次结果，但preg_match_all可以循环一个列表的匹配结果数组。

  12. cookie的设置采用函数setcookie()
    删除cookie只需将时间改为 time()-1

  13. 当我们设置了有效路径'/path'的时候，不在当前路径的时候则看不到当前cookie。

  14. session是将用户的会话数据存储在服务端，没有大小限制，通过一个session_id进行用户识别，PHP默认情况下session id是通过cookie来保存的，因此从某种程度上来说，seesion依赖于cookie。但这不是绝对的，session id也可以通过参数来实现，只要能将session id传递到服务端进行识别的机制都可以使用session。

  15. 在PHP中使用session非常简单，先执行session_start方法开启session，然后通过全局变量$_SESSION进行session的读写。session会自动的对要设置的值进行encode与decode，因此session可以支持任意数据类型，包括数据与对象等。

  16. session可以方便的存取多种数据类型，而cookie只支持字符串类型，同时对于一些安全性比较高的数据，cookie需要进行格式化与加密存储，而session存储在服务端则安全性较高。

  17. 判断文件是否存在:如果只是判断文件存在，使file_exists就行，file_exists不仅可以判断文件是否存在，同时也可以判断目录是否存在，从函数名可以看出，is_file是确切的判断给定的路径是否是一个文件。

  18. 获取文件的属性:

    • fileowner：获得文件的所有者
    • filectime：获取文件的创建时间
    • filemtime：获取文件的修改时间
    • fileatime：获取文件的访问时间
    • filesize : 可以取得文件的大小

  19. PHP中获取时间的方法:

    • time() 获取服务器当前时间的时间戳(时间戳表示从 1970年1月1日 00:00:00 到当前时间的秒数之和。)
    • date(时间戳的格式, 规定时间戳【默认是当前的日期和时间，可选】) 用来获取当时的日期。
    • strtotime()获取某个日期的时间戳，或获取某个时间的时间戳。 该函数预期接受一个包含美国英语日期格式的字符串并尝试将其解析为 Unix 时间戳。格式为strtotime(要解析的时间字符串, 计算返回值的时间戳【默认是当前的时间，可选】)。成功则返回时间戳，否则返回FALSE。
    • gmdate 函数能格式化一个GMT的日期和时间，返回的是格林威治标准时（GMT）。(中国地区的时间为GMT+8，即该函数返回的时间比date函数返回的时间晚了8个小时)。

  23. PHP中的异常处理程序:

    • Try - 使用异常的函数应该位于 “try” 代码块内。如果没有触发异常，则代码将照常继续执行。但是如果异常被触发，会抛出一个异常。
    • Throw - 这里规定如何触发异常。注意：每一个 “throw” 必须对应至少一个 “catch”，当然可以对应多个”catch”
    • Catch - “catch” 代码块会捕获异常，并创建一个包含异常信息的对象。
